# Mindy Coding Workspace

Kho lưu trữ **dữ liệu và tài liệu dạy học** dành cho quá trình tự học, giảng dạy và thực hành các kỹ năng lập trình. Nội dung được tổ chức theo từng chủ đề để người học có thể theo dõi lộ trình, học lý thuyết và thực hành từng bước.

## Nội dung hiện có

### Git

Thư mục [`docs/Git`](docs/Git) chứa bộ tài liệu nhập môn Git và GitHub dành cho người mới, đặc biệt phù hợp với môi trường Windows và PowerShell.

| Tài liệu | Nội dung chính |
| --- | --- |
| [`000 - Git Roadmap.md`](docs/Git/000%20-%20Git%20Roadmap.md) | Lộ trình học Git từ cơ bản đến sử dụng thành thạo |
| [`001 - Foundation setup config git.md`](docs/Git/001%20-%20Foundation%20setup%20config%20git.md) | Cài đặt Git, cấu hình ban đầu và phân biệt Git với GitHub |
| [`002 - Git Basics.md`](docs/Git/002%20-%20Git%20Basics.md) | Repository, working directory, staging area, commit và quy trình làm việc cơ bản |
| [`003 - Effective Commits.md`](docs/Git/003%20-%20Effective%20Commits.md) | Cách chia nhỏ thay đổi và viết commit message rõ ràng, hiệu quả |

## Lộ trình học gợi ý

1. Đọc **Git Roadmap** để nắm mục tiêu và thứ tự kiến thức.
2. Hoàn thành phần **Foundation** để cài đặt và cấu hình môi trường.
3. Học **Git Basics** và thực hành quy trình `status → diff → add → commit`.
4. Đọc **Effective Commits** để xây dựng lịch sử thay đổi dễ hiểu và dễ review.

## Đối tượng sử dụng

- Người mới bắt đầu học Git và GitHub.
- Người học lập trình cần tài liệu thực hành theo lộ trình.
- Người hướng dẫn muốn tái sử dụng nội dung cho buổi học hoặc bài tập.

## Cách sử dụng

Clone repository về máy:

```bash
git clone https://github.com/MJoy1410/mindy-workspace.git
cd mindy-workspace
```

Sau đó mở các tài liệu trong `docs/Git` bằng trình soạn thảo hỗ trợ Markdown, chẳng hạn Visual Studio Code.

## Định hướng phát triển

Kho dữ liệu sẽ tiếp tục được bổ sung thêm bài học, bài tập thực hành, ví dụ và tài nguyên tham khảo theo từng chủ đề lập trình.

## Đóng góp

Nếu muốn bổ sung hoặc chỉnh sửa tài liệu, hãy tạo một nhánh mới, commit thay đổi với thông điệp rõ ràng, sau đó mở pull request để review.
